# GitLab MCP Server 实现计划

## 项目概述
创建一个功能完整的GitLab MCP Server，支持多种传输协议（Stdio、SSE、Streamable），提供安全的GitLab操作能力。

## 项目结构
```
gitlab-mcp-server/
├── cmd/
│   └── server/
│       └── main.go           # 主程序入口
├── internal/
│   ├── config/
│   │   └── config.go         # 配置加载和管理
│   ├── gitlab/
│   │   ├── client.go         # GitLab API客户端封装
│   │   ├── project.go        # 项目相关操作
│   │   ├── mr.go            # MR相关操作
│   │   ├── file.go          # 文件操作
│   │   └── commit.go        # 提交相关操作
│   ├── mcp/
│   │   ├── server.go        # MCP服务器核心逻辑
│   │   ├── tools.go         # 工具定义和注册
│   │   ├── transport.go     # 传输方式抽象层
│   │   ├── stdio.go         # Stdio传输实现
│   │   ├── sse.go           # SSE传输实现
│   │   └── streamable.go    # Streamable传输实现
│   └── security/
│       └── validator.go     # 安全验证器（限制危险操作）

├── go.mod                   # Go模块文件

└── README.md               # 项目文档
```

## 实现步骤

### 第1阶段：基础框架（Day 1）
1. **初始化项目**
   - 创建go.mod，设置Go 1.23.8
   - 添加MCP依赖（github.com/metoro-io/mcp-golang）
   - 添加GitLab依赖（github.com/xanzy/go-gitlab）
   - 设置标准项目结构

2. **配置管理模块**
   - 定义Config结构体（GitLab host、token、传输类型等）
   - 实现config.yaml加载逻辑
   - 支持环境变量覆盖配置

### 第2阶段：MCP核心（Day 2）
3. **MCP协议实现**
   - 实现Message、Request、Response结构
   - 实现Tools列表和调用机制
   - 实现JSON-RPC 2.0协议处理

4. **Transport层实现**
   - **Stdio传输**：标准输入输出通信
   - **SSE传输**：HTTP长连接Server-Sent Events
   - **Streamable传输**：基于TCP的长连接流式传输
   - 统一的Transport接口设计

### 第3阶段：GitLab功能（Day 2-3）
5. **GitLab客户端封装**
   - 使用go-gitlab库创建客户端
   - 实现认证和请求处理
   - 统一的错误处理

6. **安全工具实现（允许的操作）**
   - `gitlab_list_projects` - 列出项目（只读）
   - `gitlab_get_project` - 获取项目详情（只读）
   - `gitlab_list_mrs` - 列出MR（只读）
   - `gitlab_get_mr_details` - 获取MR详情（只读）
   - `gitlab_get_file_content` - 获取文件内容（只读）
   - `gitlab_list_commits` - 列出提交记录（只读）
   - `gitlab_list_pipelines` - 列出流水线（只读）
   - `gitlab_create_MR_note` - 创建MR评论（写入）
   - `gitlab_create_mr` - 创建新MR（写入，非master/test）

7. **安全限制实现**
   - 实现validator.go安全验证器
   - **禁止操作**：
     - 代码合并（accept_mr）
     - master分支的代码推送和修改
     - test分支的代码推送和修改
     - 删除操作（项目、分支、MR等）
     - 配置修改操作
   - 分支名称验证规则

### 第4阶段：服务器集成（Day 3）
8. **MCP服务器核心**
   - 实现请求路由和分发
   - 工具调用处理和结果返回
   - 生命周期管理（启动、停止）

9. **主入口程序**
   - 命令行参数解析（config文件、传输类型等）
   - 根据传输类型启动对应服务器
   - 信号处理和优雅关闭

### 第5阶段：部署支持（Day 4）
10. **Docker支持**
    - 创建Dockerfile（多阶段构建）
    - 优化镜像大小
    - 支持环境变量配置

11. **文档更新**
    - 完善README.md
    - 添加Docker运行说明
    - 提供使用示例

## 技术选型
| 组件 | 选择 | 理由 |
|------|------|------|
| MCP SDK | github.com/metoro-io/mcp-golang | 官方推荐，活跃维护 |
| GitLab API | github.com/xanzy/go-gitlab | 完整的GitLab API支持 |
| 配置格式 | YAML | 易于阅读和编辑 |
| HTTP框架 | 标准库 | 轻量，无额外依赖 |

## 安全策略详解
- **只读操作**：全部开放（列表、查询类）
- **写操作限制**：
  - 仅允许创建MR（source分支不能是master/test）
  - 仅允许MR评论
  - 禁止创建分支
  - 禁止推送代码
- **删除操作**：全部禁止
- **配置修改**：全部禁止
- **合并操作**：全部禁止
